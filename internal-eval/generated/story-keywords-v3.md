# 故事生成关键词 v3

基于严格版 `codex-batch-label-v2-strict-scoring` 对当前 100 条样本的结果，本版重点补强低分维度：

- 情感平均分最低，约 2.72：很多故事有事件代价，但主角的内心反应、关系压力和个人损失不够可感。
- 文笔平均分约 2.96：具体物件不少，但感官细节、动作承载情绪、自然句流畅度还不稳定。
- 逻辑低分样本约 31 条：常见问题是坏结局由外力突然落下，台词和失败代价之间缺少补救过程。
- 长度异常仍有 26 条：背景规则和二次反转仍会把故事推到 600 字以上。

## 写作控制关键词

- 短篇密度
- 单一危机
- 行动推进
- 因果闭环
- 内心反应
- 感官细节
- 威胁递进
- 坏结局倒扣
- 台词触发
- 少设定多代价

## 生成规则 v3

- 目标长度收紧到 360-480 字；超过 520 字时优先删除背景解释、旁支角色、重复心理和二次反转。
- 每段至少有一个具体动作、物件状态或感官细节，避免整段抽象说明。
- 任何主题都要写主角的个人代价或内心反应；悬疑、权谋、盗取潜入、犯罪、战争也不能只写事件推进。
- quote 前必须有可见压力递进，例如倒计时、包围、追捕、失控、门锁、毒性、警报、审判、交易窗口或关系破裂。
- quote 后必须形成“台词/行动 -> 对方反应/机制启动 -> 主角尝试补救 -> 代价落到主角”的链条。
- suffixText 不能临时新增决定胜负的大设定；决定坏结局的规则、物件或限制必须在前文已经出现。
- 避免角色名与叙述粘连，例如“林默霍华德猛地转身”；说话方和回应方必须明确区分。

## 低分主题补强

- 亲情/爱情：加入共享物件、旧承诺、保护欲、愧疚或不舍，让坏结局伤到关系本身。
- 权谋：不要只写制度博弈，要让主角在身份、亲信、证据或承诺之间付出个人代价。
- 盗取潜入：除了机关和安保，还要写主角为什么必须偷，以及失败后失去的人或承诺。
- 犯罪/悬疑：线索推进之外，需要写主角的判断压力和误判成本。
